Patrones de diseño

Los patrones de diseño empleados en la elaboración de esta app son los siguientes:

  • Adaptador(ORM - Pony)

    El empleo de una biblioteca de ORM (Object Relational Mapping) elimina la dependencia de una base de datos específica, permitiendo reutilizar el código y escribir las consultas de una forma más idomática en Python.

  • Cadena de responsabilidad(servidor web)

    Flask (así como express) utiliza este patrón para distribuir los requests HTTP entre sus endpoints. Cada ruta en el framework consiste en una función que hace match a la URL solicitada. Si hay match, se ejecuta el código correspondiente. si no, se pasa a la siguiente ruta.
    En express, es posible indicasr explícitamente si se desea terminar el procesamiento del request, o incluso, continuar en la lista de handlers, aún si se ha ejecutado el código propio.
